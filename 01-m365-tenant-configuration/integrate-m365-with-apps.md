# Integrate Microsoft 365 with Customer Engagement Apps

---

## 🔑 Key Concepts (Must Know)

### What Does Integration Mean?

Integration means extending Microsoft 365 by:

- Adding new services  
- Connecting third-party apps  
- Enhancing business capabilities  

👉 Example:
Adding Power BI, Visio, or Project to existing M365 setup

---

## 🧠 Why It Matters

Integration helps organizations:

- Improve collaboration  
- Enhance productivity  
- Support business workflows  
- Enable cross-device access  

👉 Real-world:
Companies don’t just use “core M365” — they expand it

---

## ⚙️ Ways to Integrate Apps

Microsoft 365 provides two main methods:

### 1. Purchase Services Page (MOST COMMON)

👉 Located in:
Admin Center → Billing → Purchase Services

---

### What You Can Do There

- Buy new Microsoft 365 plans  
- Add add-on services  
- Manage licenses  
- View purchase history  
- Compare products  

---

### Examples of Add-On Services

- Visio  
- Project Online  
- Power BI  
- Microsoft Stream  
- Microsoft Purview eDiscovery (Premium)

---

👉 Real-world:
You don’t upgrade the whole plan — you add only what you need

---

### 🔥 Key Concept: Add-On Licensing

Instead of upgrading all users:

- Add specific licenses to specific users  
- Saves cost  

👉 Example (INTERVIEW QUESTION):
If only IT needs Visio → assign Visio license to IT team only

---

## 📊 Product Comparison Feature

You can compare products before purchasing:

- Features  
- Pricing  
- Capabilities  
- Supported platforms  

👉 Helps choose between:
Business Standard vs Business Premium

---

## ⏳ Trial Subscriptions

- Many services offer **30-day trials**  
- Full functionality during trial  
- Access removed if not purchased  

👉 Real-world:
Test tools before committing cost

---

## 👥 Assigning Services

Two ways:

1. Purchase → assign via Users page  
2. Assign directly after purchase  

---

## 🏪 Azure Marketplace (Advanced Concept)

### What is Azure Marketplace?

A platform to:

- Discover apps  
- Try apps  
- Purchase apps  

Includes:
- Microsoft apps  
- Third-party (ISV) apps  

---

### Private Azure Marketplace (IMPORTANT)

- Admin-controlled marketplace  
- Only approved apps can be deployed  

👉 Think of it as:
“App Store with company restrictions”

---

### Key Features

- Control which apps users can install  
- Create collections of approved apps  
- Govern third-party integrations  

---

## 🔐 Why Private Marketplace Matters

- Improves security  
- Ensures compliance  
- Prevents unauthorized app usage  

---

## 🔥 Interview Questions

### Q1: How do you add new services like Visio to Microsoft 365?

👉 Answer:
By purchasing an add-on license via the Purchase Services page and assigning it to specific users.

---

### Q2: Should you upgrade to E5 if only one team needs Visio?

👉 Answer:
No. It’s more cost-effective to purchase a Visio add-on license for only the required users.

---

### Q3: What is Azure Marketplace?

👉 Answer:
A platform where organizations can discover, try, and purchase Microsoft and third-party applications.

---

### Q4: What is Private Azure Marketplace?

👉 Answer:
A controlled version of Azure Marketplace where admins restrict which apps users can deploy.

---

## 💼 Real-Life IT Tasks

- Adding licenses like Visio or Project  
- Evaluating trial services  
- Assigning add-ons to users  
- Comparing plans before purchase  
- Controlling third-party app access  
- Supporting business tool requests  

---

## ⚠️ Common Mistakes

- Upgrading entire tenant instead of using add-ons  
- Assigning licenses to all users unnecessarily  
- Not controlling third-party apps  
- Forgetting trial expiry  

---

## 🧠 Summary

- Use add-ons instead of full upgrades when possible  
- Purchase services via Admin Center  
- Use Azure Marketplace for extended integrations  
- Control apps with Private Marketplace  

👉 Smart integration = better functionality + lower cost
